# 🧠 Idea of this setup

You will have **2 tokens**:

### 🍪 1. Auth cookie (HttpOnly)

* used for login
* NOT readable by JS

### 🔐 2. CSRF token (readable)

* used only for mutating requests
* sent in header

# 🟦 BACKEND (ASP.NET Core)

## 1️⃣ Generate CSRF token + set cookie

```csharp id="csrf1"
public IActionResult GetCsrfToken()
{
    var csrfToken = Guid.NewGuid().ToString("N");

    Response.Cookies.Append("csrf_token", csrfToken, new CookieOptions
    {
        HttpOnly = false, // IMPORTANT: JS must read it
        Secure = true,
        SameSite = SameSiteMode.None,
        Expires = DateTime.UtcNow.AddHours(1)
    });

    return Ok(new { csrfToken });
}
```

---

## 2️⃣ Validate CSRF on backend

Create middleware or filter:

```csharp id="csrf2"
public class CsrfMiddleware
{
    private readonly RequestDelegate _next;

    public CsrfMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task Invoke(HttpContext context)
    {
        var method = context.Request.Method;

        if (method == "POST" || method == "PUT" || method == "DELETE")
        {
            var cookieToken = context.Request.Cookies["csrf_token"];
            context.Request.Headers.TryGetValue("X-CSRF-TOKEN", out var headerToken);

            if (string.IsNullOrEmpty(cookieToken) || cookieToken != headerToken)
            {
                context.Response.StatusCode = 403;
                await context.Response.WriteAsync("CSRF validation failed");
                return;
            }
        }

        await _next(context);
    }
}
```

function getCookie(name: string) {
  return document.cookie

    .find(row => row.startsWith(name + "="))
    ?.split("=")[1];

# 🧠 Final flow

## 1️⃣ User opens site

```
GET /csrf/token
```

## 2️⃣ Backend sets:

```
csrf_token=abc123 (NOT HttpOnly)
access_token=xyz (HttpOnly)
```

## 3️⃣ Frontend:

* reads csrf_token from cookie
* sends it in header

## 4️⃣ Backend:

* compares header vs cookie
* allows request if match

# 🔥 Important security notes

## ✔ Why CSRF token is NOT HttpOnly here

Because frontend MUST read it.

---

## ⚠️ But don’t worry

Even if attacker knows CSRF token:

* they cannot read HttpOnly auth cookie
* browser still enforces origin rules
